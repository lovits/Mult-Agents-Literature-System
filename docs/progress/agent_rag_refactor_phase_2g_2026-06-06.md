# Agent-RAG 重构 Phase 2G 进度

日期：2026-06-06

## 本阶段目标

在可复现实验清单之上增加批量实验调度能力，让一个数据集版本对应的多篇论文审计可以通过统一入口创建、绑定不可变论文版本并追踪运行结果。

## 已完成内容

1. 批量实验调度 API
   - 新增 `POST /api/experiments/{manifest_id}/paper-audits`。
   - 单次请求支持 1 至 100 个论文审计条目。
   - 每个条目只接收论文 ID、弱点列表和检索参数，不接收证据正文。
   - 批量结果保持输入顺序，返回逐项状态、公开错误码和统计计数。

2. 部分成功语义
   - 单篇论文缺失、输入范围错误或队列不可用不会回滚其他成功条目。
   - 成功创建的运行立即加入实验清单。
   - 队列投递失败但已持久化的运行也加入清单，并保留失败状态用于复现实验故障。
   - 公共响应不返回内部队列错误信息。

3. 不可变版本绑定
   - 每个批量条目复用既有持久化论文审计路径。
   - 调度时自动绑定论文当前活动版本和有序证据块 ID。
   - 清单快照可追踪每个运行使用的论文版本和运行配置。
   - 运行输入不保存证据正文。

4. 分层收敛
   - HTTP schema 负责严格请求校验和批量大小限制。
   - 内部 `PersistedPaperReviewAuditRequest` 隔离 FastAPI 输入模型与应用服务。
   - `ExperimentManifestService` 负责批量编排和清单成员关系。
   - `ReviewAuditService` 负责单运行创建、版本快照和队列投递。

## 当前限制与风险

- 批量请求当前要求调用方提供论文 ID 和弱点列表，尚未从数据集文件自动生成条目。
- 部分成功不会自动重试失败条目，需要后续增加可控重试策略。
- 尚未实现实验级指标聚合、CSV/Markdown 论文表格导出。
- 尚未将 PeerQA-XT、PeerReview Bench 和 GLM reviewer 历史结果导入统一清单。
- 当前仍为 `silver diagnostic`，不替代人工金标评估。
- 未实现前端、鉴权、多租户和对象存储。
