# Agent-RAG 重构 Phase 2F 进度

日期：2026-06-06

## 本阶段目标

建立可复现实验清单，将数据集版本、实验级配置、不可变论文版本、审计运行和报告关联起来，为后续批量实验复跑与论文结果整理提供统一记录入口。

## 已完成内容

1. 实验清单持久化
   - 新增 `experiment_manifests`。
   - 保存实验名称、数据集名称、数据集版本和实验级配置。
   - 新增 `experiment_manifest_runs`，按加入顺序关联已有运行。
   - 同一运行重复加入同一清单时保持幂等。

2. 可复现快照
   - 清单详情汇总运行 ID、论文 ID、不可变论文版本、证据来源、运行配置、运行状态和报告元数据。
   - 使用公共字段白名单。
   - 不返回原始运行输入、弱点正文、证据块 ID 列表、内部错误或报告文件路径。
   - 实验配置递归拒绝 `api_key`、token、secret、password 等敏感键。
   - 应用级验证错误响应不回显被拒绝的请求值。

3. 实验清单 API
   - `POST /api/experiments`
   - `GET /api/experiments`
   - `GET /api/experiments/{manifest_id}`
   - `POST /api/experiments/{manifest_id}/runs/{run_id}`

## 分层边界

- Repository 只负责清单、成员关系和报告查询持久化。
- `ExperimentManifestService` 负责可复现快照组装与脱敏。
- FastAPI 路由只负责严格请求校验和 404 映射。
- 清单不修改运行、论文版本或报告。

## 当前限制与风险

- 当前只关联已有运行，尚未支持通过清单批量创建和调度运行。
- 清单配置是结构化 JSON，但尚未定义统一的实验参数注册表。
- 清单不会阻止一个运行被多个实验清单引用。
- 当前仍为 `silver diagnostic`，不替代人工金标评估。
- 未实现前端、鉴权、多租户和对象存储。
